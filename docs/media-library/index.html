Media Library

The media library is the heart of Codecurate. It's where you browse, filter, and understand everything on your server.

How indexing works​

When Codecurate starts, it recursively scans your MEDIA_DIR and runs ffprobe on every supported media file. The extracted metadata — codecs, resolutions, bitrates, languages, durations, formats — is stored in MongoDB and becomes the source of truth for all browse and filter operations.

While running, the backend:

  • Watches the filesystem for new, changed, renamed, or deleted files
  • Re-scans changed files after a short debounce
  • Runs a full reconciliation every 10 minutes (configurable via MEDIA_RESCAN_INTERVAL_MINUTES) to catch anything the watcher missed

This means you can add files to your media directory at any time and Codecurate picks them up automatically.

The browse view​

The main browse page is a dense, scrollable table showing every indexed media file. Each row shows:

ColumnWhat you see
FileFile name and path
DurationRuntime in HH:MM:SS
QualityRated bitrate health — high, good, fair, low, trash, or unknown
ResolutionEffective vertical resolution (e.g. 1080p)
BitrateTotal bitrate in kbps
Video codecsAll video codecs in the file
Audio codecsAll audio codecs in the file
LanguagesAudio and subtitle languages
FormatContainer format (.mkv, .mp4, etc.)
SizeFile size, human-readable
CreatedFilesystem creation date

Virtual scrolling​

The table uses virtualized rendering — it only paints the rows currently visible in your viewport. This keeps things fast even with 100,000+ entries.

Sorting and column control​

Click any column header to sort. Drag columns to reorder them. Toggle column visibility from the column menu. The table remembers your sort state in the URL, so you can share or bookmark a specific view.

Batch actions​

Select multiple rows using checkboxes. Once you have a selection, you can run a command preset on all selected files at once. Select an entire season, pick your AV1 preset, and queue them all up in one click.

Filtering​

The insights panel on the browse page shows distributions across your collection and doubles as a filter bar.

Filter categories​

FilterExamples
Video codecAV1, H.264, H.265, VP9, MPEG-4
Audio codecOpus, AAC, AC-3, FLAC, DTS, TrueHD
Resolution2160p+, 1080p, 720p, 576p, SD, Unknown
Bitrate healthhigh, good, fair, low, trash, unknown
Format.mkv, .mp4, .avi, .ts
Audio languageAny detected language, normalized (e.g. eng→en, deu→de)
Subtitle languageSame normalization as audio

Combining filters​

Filters stack. Select video_codec:H.264 AND resolution:1080p AND audio_codec:AC-3 and the table shows only 1080p H.264 files with AC-3 audio. Each filter shows a count and percentage, so you always know how many files match before you click.

Filter bars​

Each filter category shows a horizontal bar chart — the width of each segment represents its share of your library. This makes it immediately obvious that, say, 60% of your audio tracks are still AC-3 while only 12% are Opus.

Statistics​

The statistics endpoint (also accessible from the frontend — currently routed to browse) returns top-level counts and percentages:

  • Total file count
  • Codec distribution (e.g. "34% AV1, 41% H.264, 15% H.265")
  • Resolution breakdown
  • Format share
  • Audio codec usage

These numbers update live as your library changes.

Search​

The search bar does a case-insensitive match against file name and path. Type a partial title and the table filters instantly. Combine search with insight filters for precision — e.g. search "Star Wars" filtered to format:.mkv.

Folder browser​

The sidebar includes a folder tree of your media directory. Click a folder to narrow the browse view to only files within that subtree. Useful when your media is organized as Movies/ and Shows/ — quickly jump to one branch.

File operations​

Every row in the table has a context menu with actions:

Rename​

Rename a file on disk. The backend updates the stored path in MongoDB and any compare history referencing that path. If a file already exists at the target path, the rename is rejected.

Delete​

Deletes the file from disk and removes its MongoDB entry. This is destructive and permanent — there's a confirmation dialog.

Download​

Generates a short-lived download ticket, then streams the file. The download endpoint (GET /api/media/download) requires the ticket, not a JWT, so download links can be shared or opened in a new tab.

Upload​

Upload new media files into the current browsed directory. The backend accepts multipart uploads up to 50 GB by default (configurable via MEDIA_UPLOAD_MAX_BYTES). Uploaded files are indexed immediately after the upload completes.

Compare selection​

Select two files for side-by-side comparison. The selection is cached in local storage and carried over to the Compare page automatically.

Run preset​

Pick a command preset from the context menu to queue a conversion job on a single file. For batch operations, select multiple rows first, then choose a preset.

Add to obtain list​

Quickly add a file to your obtain list using its filename for auto-matching against TMDB.

API endpoints at a glance​

EndpointWhat it does
GET /api/mediaPaginated listing with search, filters, sorting
GET /api/media/insightsDistributions for filter panels
GET /api/media/statisticsTop-level counts and shares
POST /api/media/uploadMultipart upload into MEDIA_DIR
POST /api/media/renameRename a file on disk
DELETE /api/mediaDelete a file from disk
POST /api/media/download-ticketIssue a short-lived download ticket
GET /api/media/downloadStream a file with a valid ticket
GET /api/filesystemRecursive folder tree
