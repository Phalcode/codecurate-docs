# Codecurate Documentation

This site is organized so backend and frontend documentation can grow independently without having to reshuffle the navigation later.

Both the backend and frontend sections are now documented from the current codebases. The split is still intentional: backend pages remain the source of truth for API and operational behavior, while frontend pages focus on application structure, auth flow, API integration, and user workflows.

## Documentation map

- [Backend overview](/docs/backend/overview) explains the runtime architecture, storage model, and major services.
- [Getting started](/docs/backend/getting-started) covers prerequisites, configuration, local development, and Docker notes.
- [Authentication and access](/docs/backend/authentication-and-access) explains JWT requirements, client restrictions, CORS, and secure downloads.
- [Media library](/docs/backend/media-library) documents indexing, querying, uploads, renames, deletes, and filter syntax.
- [Automation and FFmpeg](/docs/backend/automation-and-ffmpeg) covers command presets, job lifecycle, worker modes, and queue behavior.
- [Compare and obtain](/docs/backend/compare-and-obtain) documents metadata compare, VMAF workflows, history, and TMDB-backed obtain flows.
- [API reference](/docs/backend/api-reference) is the endpoint-by-endpoint quick lookup.
- [Frontend overview](/docs/frontend/overview) explains the React application shell, route map, and major feature areas.
- [Frontend getting started](/docs/frontend/getting-started) covers prerequisites, scripts, environment variables, and OpenAPI generation.
- [Frontend authentication and API](/docs/frontend/authentication-and-api) documents OIDC, silent renew, unauthorized retry behavior, and generated client usage.
- [Frontend workflows](/docs/frontend/workflows) maps the main pages to backend capabilities and user flows.

## Start here

If you are deploying or developing the backend first, read these pages in order:

1. [Backend overview](/docs/backend/overview)
2. [Getting started](/docs/backend/getting-started)
3. [Authentication and access](/docs/backend/authentication-and-access)
4. [Media library](/docs/backend/media-library)

If you are integrating a client application, the fastest path is:

1. [Authentication and access](/docs/backend/authentication-and-access)
2. [API reference](/docs/backend/api-reference)
3. [Compare and obtain](/docs/backend/compare-and-obtain)

## Current backend scope

- ASP.NET Core API hosted on port `5000`
- MongoDB-backed metadata, queue, compare, and obtain storage
- Background media indexing with file watching and periodic reconciliation
- FFmpeg command preset execution with local and Docker worker modes
- Metadata compare and queued VMAF calculation workflows
- TMDB-backed search, translations, titles, and obtain-list automation

## Notes for future frontend docs

The frontend section is now documented at a high level and already has room for deeper additions such as:

- application architecture and page map
- authentication flow and token handling
- API client usage against the backend documented here
- media, compare, command, and obtain user workflows

When frontend docs are added, the backend pages can remain stable and keep acting as the source of truth for the API and operational behavior.
